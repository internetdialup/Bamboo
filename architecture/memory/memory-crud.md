# CRUD Create Read Update Delete/Destroy

CRUD is an essential practice for any software development project that needs to manage its memory and context. It is the basic loop that keeps memory alive instead of stale.

- CREATE: Add memory when it matters. New Knob, new rule, new architecture change, new handoff.
- READ: Retrieve the right memory before acting. Do not pretend the context is loaded when it is still cold.
- UPDATE: Refresh memory when the project changes. Stale memory is worse than missing memory.
- DELETE / DESTROY: Remove or archive memory that is wrong, duplicated, obsolete, or actively causing drift.

CRUD should be performed on ADM and RAG accordingly. ADM keeps episodic chapters healthy. RAG keeps semantic and procedural memory clean. The Watchdog should notice when either one starts rotting.
